

깃허브 서브모듈은 메인 프로젝트에 다른 깃 저장소를 포함하여 관리하는 방법입니다. 서브모듈을 추가하려면 `git submodule add` 명령어를 사용합니다. 이를 통해 서브모듈이 메인 프로젝트에 추가되고 `.gitmodules` 파일에 서브모듈 정보가 기록됩니다. 

1. 메인 프로젝트 준비:
- 메인 프로젝트를 깃 저장소로 초기화합니다.
- 메인 프로젝트를 깃허브에 업로드합니다.

2. 서브모듈 추가:
- 서브모듈로 사용할 깃 저장소를 메인 프로젝트의 특정 디렉토리에 복사합니다.
- 메인 프로젝트의 루트 디렉토리에서 다음 명령어를 실행하여 서브모듈을 추가합니다.

```shell
#코드
git submodule add <서브모듈 URL> <서브모듈 디렉토리명>

#예시
git submodule add git@github.com:user/submodule-repo.git submodule_dir
```
- `.gitmodules` 파일에 서브모듈 정보가 추가된 것을 확인할 수 있습니다. 

3. 서브모듈 초기화 및 업데이트:
- 서브모듈을 포함하는 메인 프로젝트를 클론할 때, `git clone --recurse-submodules <URL>` 명령어를 사용하여 서브모듈을 자동으로 초기화하고 업데이트할 수 있습니다. 
- 이미 서브모듈이 있는 메인 프로젝트를 클론한 후, `git submodule init` 명령어로 서브모듈을 초기화하고, `git submodule update` 명령어로 업데이트합니다. 

4. 서브모듈 업데이트:
- 서브모듈 저장소의 변경 사항을 메인 프로젝트에 반영하려면, 서브모듈 디렉토리로 이동하여 `git pull` 명령어를 실행하거나, 메인 프로젝트의 루트 디렉토리에서 git submodule update --remote 명령어를 실행합니다. 

5. 서브모듈 관리:
- `git submodule status` 명령어를 사용하여 서브모듈의 상태를 확인할 수 있습니다.
- 서브모듈의 변경 사항을 커밋할 때, 서브모듈 디렉토리에서 커밋하는 것이 아니라, 메인 프로젝트의 루트 디렉토리에서 커밋해야 합니다. 
- 서브모듈을 제거하려면, 먼저 서브모듈 디렉토리를 삭제하고, `.gitmodules` 파일에서 해당 서브모듈 정보를 삭제한 후, `git rm` 명령어를 사용하여 삭제해야 합니다. 

참고:
- 서브모듈을 사용하는 메인 프로젝트를 클론할 때, 서브모듈을 포함하여 클론하려면 git clone --recursive <URL> 명령어를 사용해야 합니다. 
- 서브모듈에 변경 사항을 적용하려면, 서브모듈 디렉토리에서 변경 사항을 커밋하고, 메인 프로젝트에서 서브모듈을 업데이트해야 합니다.
